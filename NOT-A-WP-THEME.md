# Repository notes — what ships and what does not

This directory **is** the Colophon WordPress theme: `style.css`, `functions.php`,
`theme.json`, templates, parts, and patterns are all present and the theme is
installable as-is. It is also the canonical source for the CORE/SKIN tooling that
generates the rest of the Colophon collection, so the repository carries developer
files that are **not** part of the distributed theme.

## What ships to WordPress.org

The distributed zip is the theme minus the developer tooling. Exclusions are
declared in `.distignore` and enforced by the build pipeline before packaging:

- `inc/github-updater.php` — a self-updater, disallowed in WordPress.org
  submissions. `functions.php` loads it only when the file exists, so its absence
  in the distributed zip is a no-op.
- `inc/cli.php` — the `wp colophon` developer commands.
- `colophon`, `colophon.json`, `colophon.manifest.json` — the CLI and its manifests.
- `preview/`, `*.sh`, `bin/` — the static preview system and shell tooling.
- `PILLARS.md`, this file, and the version-control artefacts.

## The Colophon collection

This theme is the documented foundation for a collection of niche editorial themes
— Kern, Masthead, Parcel, and Wake — each generated by applying its own
`inc/skin.php` to these CORE files.

| Theme    | Target audience                            |
|----------|--------------------------------------------|
| Kern     | Type foundries, brand consultancies        |
| Masthead | Digital publishers, newspapers             |
| Parcel   | DTC subscription brands                     |
| Wake     | Maritime journals, sailing clubs           |

## CORE/SKIN split

The Colophon CLI (`colophon sync`) owns the CORE files and never overwrites the
single SKIN file:

- `inc/bootstrap.php` — namespace + identity constants (the one re-prefix point)
- `inc/setup.php` — theme supports, i18n, navigation, a11y scaffolding
- `inc/assets.php` — stylesheet enqueue + font preload
- `inc/bindings.php` — copyright year + footer credit block bindings
- `inc/admin.php` — WordPress.org-compliant Get-started page
- `assets/css/core/reset.css` — cascade contract + design-agnostic reset
- `assets/css/core/base.css` — WCAG scaffolding + semantic token contract

`inc/skin.php` is the one file the CLI never overwrites — every theme-specific
decision (image crops, fonts, block styles, onboarding copy) lives there.
